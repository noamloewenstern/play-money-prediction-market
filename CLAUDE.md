# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

Play Money is a prediction market platform. Turborepo monorepo with Next.js 14 (App Router), PostgreSQL/Prisma, SWR, and shadcn/ui. npm workspaces, no path aliases — cross-package imports use `@play-money/*` package names.

## Commands

```bash
npm run dev              # Start all dev servers (web:3000, api:3001, storybook:6006, db-studio:5555)
npm run build            # Build all apps
npm run lint             # ESLint across monorepo
npm run format           # Prettier check
npm run format:fix       # Prettier fix
npm run type-check       # TypeScript checking
npm run test             # Run all Jest tests
npm run test:watch       # Jest watch mode
npx jest path/to/file.test.ts  # Run a single test file

# Database (PostgreSQL + Prisma)
npm run db:generate      # Generate Prisma client & Zod types (MUST run after schema changes)
npm run db:push          # Sync schema changes to DB
npm run db:reset         # Reset & reseed database
npm run db:script -- <name>  # Run one-off script from packages/database/scripts/

# Docker
./scripts/docker-start.sh          # Build and start everything (auto-creates .env)
./scripts/docker-start.sh --skip-build  # Reuse existing images
docker compose logs -f              # Stream all logs
docker compose down -v              # Stop and wipe database
```

All root scripts use `dotenv-cli` to load `.env` automatically.

## Architecture

**Packages have no JS build step** — raw `.ts` files consumed directly by app bundlers via workspace symlinks. Only `apps/*` and `packages/ui` CSS actually build.

**Build order**: `db:generate` MUST complete first (generates Prisma client + Zod types). Then packages build in dependency order, then apps.

**Feature package pattern**: Each package has `lib/` (server logic, one function per file), `components/`, `context/`, `types.ts`. No index.ts barrel — consumers import sub-paths directly: `@play-money/users/lib/getUserById`.

**Key dependency chain**: `apps/*` → `markets` → `finance` + `notifications` + `quests` → `database` → (Prisma).

## Gotchas and Non-Obvious Patterns

- **Prisma `omit` strips `email`/`emailVerified`/`passwordHash` globally** from User queries via Zod schema annotations (`/// @zod.custom.omit`). You won't see these fields unless you explicitly `select` them.
- **Custom ESLint rule `enforce-protection-of-sensitive-fields`** blocks returning `email` in `NextResponse.json()`. This is intentional — PII protection.
- **Never edit `packages/database/zod/`** — auto-generated by `zod-prisma-types` on `db:generate`. Your edits will be overwritten.
- **All API routes MUST have `export const dynamic = 'force-dynamic'`** — without it, Next.js may cache API responses and return stale data.
- **Use `type` not `interface`** — ESLint enforces `consistent-type-definitions: type`.
- **Use `Array<T>` not `T[]`** — ESLint enforces `array-type: generic`.
- **`apiHandler` is isomorphic** — on server reads `next/headers` cookies; on client uses `credentials: 'include'`. On server in Docker, it swaps `NEXT_PUBLIC_API_URL` for `INTERNAL_API_URL` to reach the API container directly.
- **SWR invalidation uses exported path constants** — call `mutate(MY_BALANCE_PATH)`, not arbitrary strings. Constants in `packages/api-helpers/client/hooks.ts`.
- **SSR cache invalidation uses `revalidateTag()`** in inline Server Actions within page files, passed down to client components as props.
- **Auth is per-route, not middleware** — each protected route calls `getAuthUser(req)` manually at the top. Returns `userId` or `null`.
- **Auth supports API keys** — `getAuthUser` checks `x-api-key` header as fallback after session cookie.
- **Finance calculations must use `Decimal.js`** — never use native JS `number` for currency or shares. All amounts flow through `new Decimal()`.
- **`executeTransaction` uses Prisma `$transaction` with 10s timeout** — `maxWait: 5000, timeout: 10000`. Long-running logic inside will fail.
- **TransactionEntry is double-entry accounting** — every entry has `fromAccountId` and `toAccountId`. Balance changes across a transaction must net to zero.
- **No pre-commit hooks** — CI is the only gate. Run `format`, `lint`, `type-check` locally before pushing.
- **Domain error classes** per package (e.g., `UserNotFoundError`, `MarketNotFoundError`) — catch these in route handlers and map to HTTP status codes.
- **Mock factories return plain objects** — they satisfy TypeScript types but don't have Prisma model methods. Don't call `.save()` or similar on mocks.
- **Custom Jest matchers** — `toBeCloseToDecimal()` and `expect.closeToDecimal()` available in all tests for Decimal.js comparisons.

## API Patterns

**Route structure**: `apps/api/app/api/v1/<resource>/route.ts` + co-located `schema.ts`.

**Response shapes**:
```ts
NextResponse.json({ data: market })                                          // single
NextResponse.json({ data: items, pageInfo: { hasNextPage, endCursor, total } }) // paginated
NextResponse.json({ error: 'message' }, { status: 4xx })                     // error
```

**Validation**: Every route has `schema.ts` exporting Zod schemas per method (`schema.get.parameters`, `schema.post.requestBody`). Return type is `SchemaResponse<typeof schema.get.responses>`. Schemas reuse `MarketSchema`, `UserSchema` etc. from `@play-money/database` (auto-generated Zod types).

**Auth pattern**: `const userId = await getAuthUser(req)` at the top of protected handlers.

## Frontend Patterns

**Data fetching**: SWR hooks in `packages/api-helpers/client/hooks.ts`. Mutations call `apiHandler` functions directly, then `mutate()` path constants to invalidate.

**State**: No global state library. React Context per domain (`UserContext`, `SidebarContext`, `ReferralContext`), SWR for server state.

**Components**: PascalCase files. Suffixes: `Page`, `Form`, `Dialog`. Feature components live in their package's `components/` directory.

**Forms**: Always React Hook Form + `zodResolver` + shadcn `<Form>` components. Form persistence via `usePersistForm` from `@play-money/ui`.

**UI imports**: `import { Button } from '@play-money/ui/button'` — each component has its own export path.

## Code Style

- Prettier: single quotes, no semicolons, trailing commas (es5), print width 120
- Import sorting: third-party → `@play-money/*` → local (via `@trivago/prettier-plugin-sort-imports`)
- Tailwind class sorting via `prettier-plugin-tailwindcss`
- ESLint extends `@vercel/style-guide` + `eslint-config-turbo`

## Testing Patterns

- Jest + ts-jest, test environment: node, tests colocated as `<filename>.test.ts`
- **DB mock**: Global setup via `jest-mock-extended` — every test gets a deep-mocked Prisma client. Reset before each test via `mockReset(dbMock)`.
- **Mock factories**: `@play-money/database/mocks` provides `mockUser()`, `mockMarket()`, `mockAccount()`, `mockBalance()`, `mockMarketOption()`, `mockTransactionEntry()` etc.
- **Function mocking**: `jest.mock('./getMarketAmmAccount')` then `jest.mocked(fn).mockResolvedValue(...)`.
